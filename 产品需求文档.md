# 窗口吸附布局管理器 — 产品需求文档 (PRD)

**项目代号**：WinLayout  
**版本**：1.0  
**目标平台**：Windows 10/11 (x64)  
**技术栈**：C# + WPF，Windows 原生 API  
**安装与运行**：MSIX 打包或独立 EXE，支持开机自启并常驻系统托盘

---

## 1. 产品概述
一款 Windows 桌面工具，允许用户自定义窗口布局（如 2/3/4/5 个窗口的不同大小比例组合）。在拖拽窗口时，**按住修饰键**即可浮现当前屏幕所绑定的布局区域，鼠标所在区域将被高亮。松开鼠标后，窗口自动缩放并填充至高亮区域，实现快速、规范的窗口摆放。

该工具特别针对多屏幕场景设计，每台显示器拥有独立布局配置；外接显示器断开后布局暂存，重连后自动恢复。同时支持**快速切换布局**，区域内的窗口自动重排以适应新布局，并与**虚拟桌面**深度结合（不同虚拟桌面可拥有独立布局）。

## 2. 核心功能需求

### F1. 布局管理

#### F1.1 布局创建与编辑
- 提供**预设模板快速修改**模式：
  - 内置常见模板：左右二分、上下二分、左主右辅、三等分、四等分、2x2网格、主窗+右侧两小窗、主窗+底部两小窗等。
  - 在预设模板基础上，用户可拖动分割线，精确调整各区域的比例（百分比/像素）。
  - 支持添加/删除分割线，从模板衍生出自定义布局。
- 每个布局须命名（如“工作编码”、“股票看盘”）。
- 布局编辑器以窗口形式独立运行，可从托盘菜单或设置中打开。

#### F1.2 布局与显示器绑定
- **每台显示器独立绑定布局**：主屏可使用“左主右辅”，副屏可使用“上下二分”。
- 显示器识别方式：使用 EDID 或设备 ID 作为唯一标识符。
- 行为：
  - 外接副屏时，默认分配一个初始布局（如“二分”），用户可修改。
  - 外接屏断开：对应布局**暂存**，窗口不再受该布局约束。
  - 重新接入相同显示器时，**自动恢复**其专属布局及编辑设置。

#### F1.3 布局切换
- 系统托盘图标的右键菜单中列出当前焦点屏幕的所有布局。
- 单击某个布局即可**切换**：
  - 该屏幕上所有**当前已被布局管理的窗口**（即已在某个区域内的窗口）会根据新布局的区域定义自动调整大小和位置。
  - 窗口 → 区域的映射规则：保持原先区域序号映射（如原区域1的窗口填入新布局的区域1）；若新布局区域数量变化，则多余的窗口平铺在屏幕剩余空间，或按首次可用映射。
  - 未被布局管理的自由窗口不受影响。
- 快捷键支持：`Win + 数字` 直接切换到对应序号布局（可配置）。

### F2. 拖拽吸附交互

#### F2.1 触发条件
- **修饰键拖拽**：仅当用户按住指定修饰键（默认 `Shift`）拖拽窗口时，布局区域才浮现。
- 修饰键可在设置中更改为 `Ctrl`、`Alt` 或组合键。
- **移动阈值**：窗口被拖拽移动超过默认阈值（10px）后叠加层才显示，避免短促点击导致闪烁。阈值可在设置中调整。
- 不按修饰键拖拽 → 完全标准的 Windows 窗口移动，零干扰。
- **拖拽检测机制**：使用 `SetWinEventHook` 监听 `EVENT_SYSTEM_MOVESIZESTART` / `EVENT_SYSTEM_MOVESIZEEND` 事件，与系统拖拽行为同步，避免轮询或低级钩子带来的开销和延迟。

#### F2.2 区域显示与高亮
- 修饰键按下并拖拽窗口时，当前屏幕叠加显示半透明层：
  - **区域高亮**：鼠标光标所在的布局区域用系统主题色填充（半透明，约 30%~40% 透明度），并带有 2px 边框。
  - **其余区域暗化**：非聚焦区域叠加黑色半透明 (50%)，形成类似 Windows 11 贴靠布局预览的视觉层次。
  - 同时可在区域中央显示区域标签（例如“1”），字体颜色白色，可选显示/隐藏。
- 高亮判定基准：**鼠标光标位置**（光标位于哪个区域，就高亮哪个区域）。
- 刷新率：跟随鼠标实时移动，无明显延迟（建议至少 60fps 渲染）。
- **叠加层实现**：采用 WPF 全屏透明 Topmost 窗口 + 点击穿透（`WS_EX_TRANSPARENT`），叠加层仅负责视觉展示，不拦截鼠标事件。区域命中判定通过窗口坐标与布局区域数据进行计算，不依赖叠加层控件的命中测试。

#### F2.3 松手行为
- 松开鼠标左键（结束拖拽）：
  - 窗口**自动调整大小和位置**，精确填充整个高亮区域（含边距，默认 0px，可由布局模板定义内边距）。
  - **保持窗口 Z 序**：不改变其前后遮挡关系。
  - 如果窗口原本最大化，先取消最大化再填充。
- **吸附行为**：松手后窗口立即填充至目标区域，不添加动画过渡。
- **区域占用处理**：
  - **默认替换**：若目标区域已被另一个窗口占据，则原窗口被”挤”出，变为自由窗口（恢复原有大小并在当前屏幕居中显示）。
  - **堆叠模式**：若在松开时额外按住 `Ctrl` 键（可配置），则两个窗口**堆叠/并排**在该区域内（如选项卡或均分区域），具体堆叠模式由布局规则决定（第一版可简单上下或左右均分该区域）。

#### F2.4 跨屏幕拖拽
- 当窗口从屏幕 A 拖到屏幕 B：
  - **切换时机**：被拖窗口的中心点进入屏幕 B 超过 50% 面积时，高亮区域和布局定义切换为屏幕 B 的专属布局。
  - 拖动过程中不提前切换，避免在边缘时频繁闪烁。

#### F2.5 不受管理的窗口
- 所有常规应用窗口默认受管理（包括 UWP、Win32）。
- 系统窗口自动排除：任务管理器、开始菜单、系统托盘弹出窗口、桌面小工具等。
- 提供**黑名单/白名单**设置：
  - 黑名单：指定进程名或窗口类名，完全忽略，拖拽时不触发布局。
  - 白名单模式：仅管理列出的应用窗口。
- 最小尺寸阈值：窗口尺寸小于 200x150px（可配置）不触发布局，避免吸附工具提示。

> **1.0 范围说明**：多窗口同时吸附（框选或多个窗口批量拖入布局）不纳入 1.0 版本。1.0 的批量窗口管理通过 F3 快速填充功能提供。

### F3. 快速填充（一键应用布局）

- 功能目标：无需拖拽，将当前屏幕所有可见窗口快速填入当前布局区域。
- 触发方式：
  - 系统托盘菜单每个布局项旁边有“应用并填充”子选项，或右键布局 → “用当前窗口填充”。
  - 可配置全局快捷键，如 `Win + Alt + 数字`。
- 执行逻辑：
  - 收集当前焦点屏幕所有未被最小化的顶级窗口。
  - 按窗口 Z 序或最近使用顺序，依次填入布局的各个区域（区域 1 → 窗口 1，区域 2 → 窗口 2...）。
  - 如果窗口数量多于区域数，多余的保持原位（自由窗口）。
  - 如果窗口数量少于区域数，部分区域留空。

### F4. 与虚拟桌面结合

- **独立布局配置**：每个虚拟桌面可以拥有不同的布局设定（当前屏幕的不同布局）。
- 实现方式：
  - 使用 Windows Virtual Desktop API 获取当前桌面 ID。
  - 存储结构：`ScreenID` → `DesktopID` → `LayoutID`（当前激活布局）及布局定义列表。
- 当用户切换虚拟桌面时，布局自动切换为该桌面绑定的布局（但不强制窗口重排，除非拖拽或手动应用）。
- 设置界面中，可配置“切换桌面时自动应用布局并重排窗口”（可选开关）。

## 3. UI/UX 设计指南

### 3.1 系统托盘图标
- 常驻托盘，提供：
  - **当前屏幕布局列表**：显示布局名称，勾选当前激活项。
  - **布局编辑器入口**：打开布局编辑器窗口。
  - **设置**：打开通用设置窗口。
  - **一键应用**：子菜单“用窗口填充当前布局”。
  - **暂停/恢复**：临时禁用窗口吸附，不退出程序。
  - **退出**。

### 3.2 布局编辑器窗口
- 主区域：代表目标显示器真实比例的预览画布。
- 默认加载预设模板，分割线可拖动。
- 工具栏：
  - 选择显示器（若有多屏）。
  - 预设模板下拉列表：选中后替换当前编辑画布。
  - “新建布局”、“保存”、“另存为”、“删除”。
  - 区域数量微调按钮。
- 分割线操作：
  - 添加分割线：点击右键“添加垂直/水平分割”。
  - 删除分割线：右键或双击。
  - 拖动分割线改变比例，显示百分比提示。
- 每个区域可显示顺序编号（1,2,3...），方便映射。

### 3.3 设置窗口
- **通用**：
  - 修饰键选择：Shift / Ctrl / Alt / 组合。
  - 堆叠修饰键选择。
  - 最小窗口尺寸阈值。
  - 开机自启开关。
  - 高亮透明度/颜色微调（使用系统主题色或自定义）。
- **黑名单/白名单**：
  - 列表方式管理应用程序（显示窗口标题和进程名），支持添加/删除。
- **虚拟桌面**：
  - “各桌面独立布局”开关。
  - “切换桌面时自动重排窗口”开关。
- **布局备份**：
  - 导出/导入所有布局（JSON 文件）。

## 4. 数据结构设计

### 4.1 布局定义文件（内部）
```json
{
  "LayoutId": "guid",
  "Name": "编码三分",
  "ScreenId": "DISPLAY1-EDID",
  "Zones": [
    {
      "Index": 1,
      "Left": 0.0,
      "Top": 0.0,
      "Width": 0.5,
      "Height": 1.0,
      "Padding": 0
    }
  ],
  "IsTemplate": false
}
```

> 说明：每个区域的 `Padding` 是唯一的间距控制字段（窗口相对区域边界的内缩像素值）。不设独立的 Gap（区域间距），相邻区域的 Padding 叠加自然形成视觉间距。

### 4.2 用户配置存储

存储路径：`%LocalAppData%\WinLayout\settings.json`

```json
{
  "ModifierKey": "Shift",
  "StackingKey": "Ctrl",
  "DragThreshold": 10,
  "MinWindowSize": { "Width": 200, "Height": 150 },
  "HighlightOpacity": 0.35,
  "DimOpacity": 0.5,
  "RunAtStartup": true,
  "PerDesktopLayout": true,
  "AutoApplyOnDesktopSwitch": false,
  "BlacklistedProcesses": ["devenv.exe"],
  "ScreenLayouts": {
    "ScreenId_1": {
      "ActiveLayoutId": "guid",
      "DesktopLayouts": {
        "DesktopId_1": "layout_guid"
      }
    }
  },
  "WindowZoneMappings": {
    "{LayoutId}": {
      "1": { "ProcessName": "Code.exe", "WindowTitle": "MyProject" },
      "2": { "ProcessName": "chrome.exe", "WindowTitle": "Stack Overflow" }
    }
  }
}
```

## 5. 非功能性需求

**性能**：区域叠加层渲染不占用高 GPU，对日常办公无感知影响。

**稳定性**：崩溃后通过 Windows `RegisterApplicationRestart` API 自动重启，布局数据采用原子写入（先写临时文件再替换）确保不损坏。

**兼容性**：支持 Windows 10 2004+ 和 Windows 11，高 DPI 适配。

**安全性**：不监听键盘输入内容，仅注册全局热键和鼠标钩子（修饰键检测），符合 Windows 安全要求。

## 6. 开发里程碑建议

### 阶段一：核心引擎

- SetWinEventHook 拖拽检测、修饰键状态判定、移动阈值逻辑。
- 全屏透明点击穿透叠加层绘制、区域高亮与暗化。
- 窗口定位与调整大小（MoveWindow / SetWindowPos），取消最大化后填充。
- 单一默认布局，无编辑器（硬编码左右二分）。

### 阶段二：布局系统

- 布局编辑器、预设模板、保存/加载布局。
- 多显示器支持、布局切换与窗口自动重排。

### 阶段三：高级特性

- 虚拟桌面独立布局。
- 黑名单/白名单。
