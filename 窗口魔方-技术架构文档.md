# 窗口魔方 (WindowCube) — 技术架构文档

## 概述

窗口魔方是一款 Windows 窗口吸附布局管理器。通过拖拽窗口标题栏并释放到半透明叠加层的分区中，可快速将窗口定位到预定义或自定义布局的对应区域。支持多显示器主/副屏独立布局、系统托盘快捷操作、自定义布局编辑器。

- **语言**: C# 12 + WPF
- **框架**: .NET 8.0 (net8.0-windows)
- **输出**: WinExe (自包含单文件发布)
- **依赖**: System.Drawing.Common 8.0.x (用于系统托盘图标)
- **作者**: iambest1

## 数据模型

### LayoutDefinition (JSON 持久化文件: layouts/{LayoutId}.json)
```
LayoutId : string (GUID)
Name     : string
ScreenId : string
Zones    : List<ZoneDefinition>
IsTemplate : bool
IsDefault  : bool
IsFavorite : bool (已弃用, 改用每屏 FavoriteLayoutIds)
```

### ZoneDefinition
```
Index   : int       // 1-based 显示用
Left    : double    // 0.0~1.0 比例坐标
Top     : double
Width   : double
Height  : double
Padding : int       // 像素内边距
```
> 所有坐标使用**相对比例**，实际像素 = 比例 × 屏幕像素 + 补偿。

### UserConfig (settings.json)
```
ModifierKey                : string  // None/Shift/Ctrl/Alt/组合
StackingKey                : string  // Shift/Ctrl/Alt
DragThreshold              : int     // 拖拽触发阈值 (px)
MinWindowSize              : { Width, Height }
HighlightOpacity / DimOpacity : double
RunAtStartup               : bool
PerDesktopLayout           : bool
AutoApplyOnDesktopSwitch   : bool
BlacklistedProcesses       : List<string>
ScreenLayouts              : Dictionary<string, ScreenLayoutConfig>

ScreenLayoutConfig:
  ActiveLayoutId   : string
  FavoriteLayoutIds : List<string>
  DesktopLayouts   : Dictionary<string, string>
```

### 预设模板 (18种)
| 类别 | 模板名 | 分区数 |
|------|--------|--------|
| 基础分割 | 左右二分, 上下二分, 左主右辅 | 2~3 |
| 等分 | 三等分, 四等分, 五等分, 六等分 | 3~6 |
| 网格 | 3×3网格, 3×2网格, 4×2网格 | 6~9 |
| 主窗+侧窗 | 主窗+右侧两小窗/三小窗, 主窗+左侧二小窗/三小窗 | 3~4 |
| 主窗+底窗 | 主窗+底部两小窗/三小窗 | 3~4 |
| 主窗+顶窗 | 主窗+顶部二小窗/三小窗 | 3~4 |

## 关键算法

### 1. 拖拽吸附流程

```
HookService (30ms轮询)
  ├→ GetAsyncKeyState(VK_LBUTTON) 检测鼠标按下
  ├→ IsModifierPressed() 检测修饰键配置
  ├→ WindowFromPoint → GetAncestor(GA_ROOT) 获取顶层窗口
  ├→ SendMessage(WM_NCHITTEST) 检测标题栏
  ├→ 移动超过阈值 → DragStarted 事件
  │   └→ OverlayService.Show(光标位置)
  │       └→ 创建 OverlayWindow 全屏透明窗口
  │       └→ 绘制所有分区 + 高亮 + 编号
  ├→ 持续移动 → DragMoved 事件
  │   └→ OverlayService.UpdateCursor()
  │       └→ 检测跨屏 → 切换 _currentScreenId
  │       └→ 命中测试: 光标是否在某个分区内?
  │       └→ 设置 LastSnapTarget
  └→ 鼠标释放 → DragEnded 事件
      └→ MainWindow.OnDragEnded
          └→ Dispatcher.BeginInvoke(Background)
              └→ WindowManager.SnapWindow(hwnd, target, stacking)
```

### 2. 窗口吸附 (SnapWindow)

```
SnapWindow(hwnd, target, stacking)
  ├→ UnmaximizeWindow
  ├→ 保存原始位置到 _originalRects (首次吸附)
  ├→ RemoveWindow(hwnd) 清除旧分区占用
  ├→ 检测目标分区是否被占用
  │   ├→ IsWindow 检查 (失效窗口清理)
  │   ├→ stacking=true → StackWindowsInZone(二等分)
  │   └→ stacking=false → DisplaceWindow (恢复原始位置或居中)
  ├→ DWM 边框补偿
  │   └→ DwmGetWindowAttribute(DWMWA_EXTENDED_FRAME_BOUNDS)
  │   └→ 计算 borderLeft/Top/Right/Bottom
  │   └→ 目标位置 -= border (补偿不可见边框)
  └→ SetWindowPos(SWP_NOZORDER | SWP_NOACTIVATE | SWP_SHOWWINDOW)
```

### 3. 分区碰撞检测 (分割线)

```
DrawSplitters:
  遍历所有分区对 (i,j), i<j
    FindSplitterEdge(zoneA, zoneB):
      垂直边: |a.Right - b.Left| < ε 或 |b.Right - a.Left| < ε
      水平边: |a.Bottom - b.Top| < ε 或 |b.Bottom - a.Top| < ε
      必须有高度/宽度重叠才返回有效边缘

OnDragMouseMove:
  ├→ 根据 _zoneA 和 _zoneB 的实际位置确定分割线坐标
  │   (不能假设 A 总是在左/上)
  ├→ 找出所有共线分区
  ├→ 计算动态有效范围 (防止越过相邻分割线)
  │   minPos = max(各分区 Left/Top + 10%)
  │   maxPos = min(各分区 origRight/Bottom - 10%)
  └→ 批量调整所有共线分区
```

### 4. 叠加层渲染

```
OverlayWindow.ShowOverlay()
  ├→ 全屏 Canvas，读取当前屏幕激活布局的分区
  ├→ 绘制 55% 黑色遮罩背景
  ├→ 每个分区: 25% 白色半透明矩形 + 灰色边框
  ├→ 高亮分区: 35% 蓝色 + 白色粗边框
  ├→ 分区编号: 中心白字
  ├→ 分割线: 遍历分区对，仅绘制真实相邻边
  └→ DPI 缩放: 物理像素 ÷ DPI 比例 → WPF 设备无关像素
```

### 5. 多屏管理

```
屏幕ID: 设备路径 + "_p"(主屏) / "_s"(副屏)
       e.g. "\\\\.\\DISPLAY1_p" / "\\\\.\\DISPLAY1_s"

每屏独立配置:
  ScreenLayouts[screenId].ActiveLayoutId    ← 活跃布局
  ScreenLayouts[screenId].FavoriteLayoutIds ← 收藏列表

OverlayService 跨屏:
  └→ 拖拽时检测窗口中心点所在屏幕
  └→ 切换 _currentScreenId
  └→ 重新加载新屏幕的布局分区
```

### 6. 布局编辑器分割线交互

```
OnSplitterMouseDown:
  ├→ Mouse.Capture(PreviewCanvas, SubTree)
  └→ 记录 _zoneA, _zoneB, _isHorizontalSplitter

OnDragMouseMove:
  ├→ 更新两分区大小
  ├→ 共线检测: 同一坐标的所有分区同步移动
  ├→ RenderPreview (释放MouseCapture → 重建视觉树 → 重新捕获)
  │   (防止 Clear + 活动捕获导致 WPF 鼠标状态损坏)
  └→ 动态边界: 不能越过相邻分割线 (至少保留 10%)

OnDragMouseUp:
  └→ Mouse.Capture(null)
```

## 持久化存储

```
%LocalAppData%\WinLayout\
├── settings.json            # UserConfig (用户配置)
└── layouts\
    ├── {guid1}.json          # 每个布局一个文件
    ├── {guid2}.json
    └── ...
```

- **原子写入**: 先写 `.tmp` 文件 → File.Move(overwrite:true)
- 配置读写无缓存，每次从磁盘加载
- 可导出 `List<LayoutDefinition>` 为单个 JSON 用于分发

## 主窗口布局 (收藏管理)

```
┌─────────────────────────────────────────────┐
│  窗口魔方 — 窗口吸附布局管理器                │
│  主屏: xxx — 副屏: xxx — 直接拖拽吸附就绪     │
│  [菜单] [打开布局编辑器]                     │
├──────────┬───────┬──────────┬───────┬────────┤
│ 全部布局  │ →←⊘   │ 收藏(主屏) │ →←⊘   │ 收藏(副屏)│
│ (list)   │       │ (list)    │       │ (list)    │
└──────────┴───────┴──────────┴───────┴────────┘
```

- 左列表: 未被任何屏幕收藏的布局
- 中列表: 主屏收藏 (双击切换主屏活跃布局)
- 右列表: 副屏收藏 (仅双屏时显示)
- 每组按钮独立操作对应屏幕

## 托盘菜单结构

```
▶ 主屏布局          ▶ 副屏布局
  ✓ 左右二分          ✓ 三等分
    上下二分            四等分
  ─────────         ─────────
  快速填充当前布局    导出布局...
  导入布局...        布局编辑器
  设置
  ─────────
  暂停吸附 / 恢复吸附
  ─────────
  语言 / Language
  ─────────
  退出
```

- ✓ 标记该屏幕当前活跃布局
- 点击子项仅切换对应屏幕的布局
- Win+1~9 热键切换当前光标所在屏幕的布局

## 已知待解决问题

1. **删除布局后主窗口 UI 不实时刷新**: 编辑器中删除活跃布局后，StatusLabel 和托盘菜单未能即时更新（`Dispatcher.BeginInvoke` 和 `Owner.RefreshAll()` 均未解决）
2. **吸附窗口后另一已吸附窗口跳出**: 已修复 `RearrangeAll` 级联驱逐 + 旧占用残留，但用户报告可能仍有未知路径触发
